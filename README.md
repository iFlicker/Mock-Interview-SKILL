# Mock-Interview

`Mock-Interview` 是一个基于真实简历进行模拟面试的 Skill。它会先读取候选人的简历和目标岗位，再以真实面试官的方式逐轮提问、追问，并在结束后给出基于本轮回答的结构化评价。你可以在当前应用中进行文字面试，也可以打开网页语音面试室，让面试官自动朗读问题并用麦克风转写回答。

它适合用来做：

- 专业一面、二面、经理面、总监面等岗位专业或业务深挖；
- 人力资源初筛、最终 HR 面、薪资谈判演练；
- 行为面试、系统设计、算法面试、项目深挖、案例分析；
- 面试前查漏补缺，或面试后生成可复盘的 HTML 报告。

## 快速开始

你可以直接这样调用：

```text
使用 $mock-interview，根据我的简历进行一场品牌营销经理专业二面。
```

如果简历在工作目录里，也可以指定路径：

```text
使用 $mock-interview，读取 ./求职材料/简历.pdf，模拟门店运营经理的案例分析面试。
```

也可以提前指定配置：

```text
使用 $mock-interview 模拟高级产品经理的经理面，7 个主题，压力值 70，重点考察跨部门协作和业务取舍。
```

## 开始前需要什么

开始正式面试前，需要满足三个条件：

1. **有效简历**
   支持 PDF、DOC、DOCX、RTF、PNG、JPG、JPEG、WEBP、TXT 和 Markdown。文件无法读取、内容为空或图片无法识别时，需要换一个可读取版本。

2. **目标职位**
   目标职位必须由简历或用户明确给出，例如“品牌营销经理”“高级产品经理”“财务分析师”。Skill 不会只根据项目经历、专业背景或技能清单猜岗位。

3. **面试配置**
   Skill 会区分面试阶段、面试官角色、面试形式和压力风格。例如“专业二面”“经理面”“最终 HR 面”“系统设计面试”“压力型薪资谈判”。如果缺少会影响执行的信息，Skill 会先询问并给出合适选项。

职位描述、公司信息、题库、历史面试记录不是必需材料；如果提供了，Skill 会纳入提问和评价。
主动寻找简历时只会先按文件名和受支持的文件类型筛选明显的求职材料，不会为了找简历而扫描工作区中所有文件的内容。历史面试记录、薪资资料等敏感材料只有在你明确提供或指定时才会读取。

## 面试体验

正式开始前，Skill 会展示一份简短的本轮配置，包括候选人类型、面试类型、主题数量或考察项数量、压力值、面试官风格、语言和重点方向，并在同一次确认中请你选择交互方式：

- **文字面试**：继续在当前应用里逐轮回答，现有流程不变；
- **语音面试**：自动打开一个本地网页聊天室，问题会自动朗读，回答可以先语音转成文字、检查修改后再发送。

你可以直接回复“按当前配置开始文字面试”或“按当前配置开始语音面试”。如果你在最初的要求里已经明确指定文字或语音，Skill 不会再重复询问；只需确认配置即可。选择文字面试不会启动网页或本地服务。

正式面试统一采用沉浸式模拟：面试过程中不即时点评、不提供参考答案，结束后再统一反馈。

正式面试开始后，会出现开始线：

```text
---------------------------- 面试开始 ----------------------------
```

开始线和结束线之间，输出会尽量保持真实面试感：不会显示题号、内部主题切换、已完成进度、评分意图或“这是一个追问”这类说明。

结束时会出现：

```text
---------------------------- 面试结束 ----------------------------
```

然后才会给出评价。

上述长分界线只用于当前 Agent App。网页语音面试室使用简洁的“面试开始”和“面试结束”，不会显示长横线，也不会重复显示结束提示。

正式面试不会机械照读题库。Skill 会根据当前回答判断事实、责任边界、原理、取舍、异常、验证和结果等关键证据是否充分，优先追问最影响岗位判断的缺口。每个问题输出前还会检查是否单一、相关、非重复、可评分、不诱导且没有虚假前提。

为了让逐轮对话更接近真实面试官，Skill 会在有必要时用一个短句承接你刚才实际表达的事实、边界或判断，再提出一个主要问题；没有有意义的承接内容时会直接发问，不会机械重复“好的”“明白了”。主题变化也会通过回答中的信息自然连接，而不是宣布内部流程。

深挖会根据当前证据缺口选择不同挑战动作：从澄清、具体化和解释，逐步扩展到方案比较、证据验证、边界探测、约束变化和决策辩护。挑战不会为了显得高压固定升级，也不会连续换词重复同一种问法；复杂反事实必须建立在已经确认的背景和责任边界上。

面试官只使用你提供的材料、本轮回答、公开通用专业知识和已确认的模拟设定。简历内容会被视为待核实背景，口述内容会被视为你的陈述，不会被自动当成已经证实的事实。遇到尚未确认的推断，面试官会明确校准理解并允许你纠正，不会虚构项目内幕、公司数据或团队意图。

当回答看似前后不一致时，Skill 会先区分表述差异、上下文缺失和实质冲突。只有经过中立澄清后仍未解决的实质冲突，才会作为评价证据；不会仅凭措辞变化指控简历真实性。

## 你可以随时控制面试

面试过程中可以直接说：

- `跳过`：换一个自然的问题。
- `暂停面试`：退出面试官角色，讨论面试过程。
- `继续面试`：恢复面试。
- `调整压力到 70`：即时调整压力值，范围 1～100。
- `结束面试`：立即停止提问，并基于已有回答评价。

在网页语音面试室中，顶部和输入区也提供暂停、继续、跳过、结束和切回文字面试的入口。结束面试会先要求确认；切回文字面试不会重新开始：已确认配置、当前问题、历史回答和评分证据都会保留，后续直接在当前应用继续。

## 网页语音面试室

选择语音面试后，Skill 会先准备网页面试室并显示可点击链接，再自动打开网页，因此不会出现浏览器已经打开、当前应用稍后才提示“已尝试打开”的错位。你不需要安装 MCP、编辑配置、运行命令、选择端口或管理服务。即使浏览器没有自动打开，也可以直接点击已经提供的本地链接。

网页连接后，如果面试官还在准备开场问题，会显示旋转图标和准备状态。回答由服务确认后，消息会标记为“已发送”，随后显示“思考中...”，直到下一条正式问题到达。它们只是等待提示，不属于正式问题或评分内容。

本地服务首次遇到瞬时启动失败时会在同一次操作中自动恢复，不需要你再次输入“重试语音面试”。只有多次自动尝试都失败时，Skill 才会建议切回文字面试。

页面顶部只显示面试官信息和简洁的连接状态。看到“已连接”后即可开始，不需要试听、试说、校准或完成设备测试。

### 语音回答

1. 点击“麦克风已关闭”；
2. 浏览器首次询问时允许麦克风权限；
3. 说话时临时识别文字会单独显示，确认结果会追加到输入框；
4. 检查并修改文字；
5. 点击“发送回答”，或按 `Ctrl/Command + Enter`。

停顿不会自动提交，关闭麦克风也不会清空已经转写的文字。语音和键盘可以混合使用；只有发送成功后输入框才会清空，连接失败时回答会保留。

浏览器不支持语音识别时，面试仍可继续：直接用键盘输入，或使用 Typeless、微信输入法等系统语音输入工具。浏览器不支持朗读时，问题文字仍会正常显示。

### 问题朗读

新的面试官消息会自动朗读。每条面试官气泡都可以单独重新朗读，再次点击当前朗读按钮可以停止。浏览器能提供可靠朗读边界时，页面会高亮已朗读、正在朗读和尚未朗读的文字；边界能力不足时只显示“正在朗读”，不会显示虚假的精确进度。打开麦克风会自动停止正在进行的朗读，避免把扬声器声音识别成回答。

### 浏览器兼容性

- 推荐使用当前版本的 Chrome 或 Edge；Safari 通常支持问题朗读，但原生语音识别能力取决于系统和版本；
- Firefox 等不提供原生 `SpeechRecognition` 的浏览器仍可使用键盘和系统输入法；
- 系统声音、中文断句、朗读进度边界和麦克风权限界面由浏览器与操作系统决定；
- 云端 Agent、远程容器或 SSH 环境通常无法让你访问远端机器的 localhost，此时 Skill 会如实提示并允许切换为文字面试。

### 隐私

本地服务只允许本机访问，不会部署到公网。默认不录制或保存音频，不保存临时识别文字，也不会把完整简历发送到网页。只有你检查并发送后的回答文字会交给当前 Agent，用于本轮问题历史、证据和评分。

浏览器原生语音识别可能由浏览器厂商的在线服务处理，因此不能默认视为完全本地识别。若不希望使用该能力，可以始终使用键盘或你信任的系统输入法。

### 常见问题

**网页显示“连接中断，正在重试”怎么办？**

页面会先自动重连，再自动切换到兼容的 HTTP 通道。回答发送失败时不会清空；长时间无法恢复时可返回当前应用切换为文字面试。

**浏览器没有弹出麦克风授权？**

只有点击麦克风开关时才会请求权限。请检查地址栏附近的站点权限；没有音频输入设备时可以继续键盘回答。

**刷新网页会重新开始吗？**

不会。网页会恢复本轮已有消息和回答，历史问题不会自动重新朗读，也不会重复写入评分证据。

**停顿或暂时不回答会自动结束吗？**

不会。面试会持续等待你检查和发送回答；只有你明确确认“结束面试”时才结束。如果顶部变为“无法连接”，请返回当前应用继续或切换为文字面试。

**回答发送后为什么输入区暂时不可用？**

这表示当前回答已经成功提交，面试官正在处理。新的问题或重新表述到达后，输入区会自动恢复；不会要求你对已经提交的问题重复发送。

**等待回答时为什么看不到重复命令？**

语音面试会在后台使用一个持续等待任务接收回答。短轮询和重连由它内部完成，同一面试室不会同时启动多个等待任务，也不需要你处理任何命令。

**自动打开浏览器失败怎么办？**

点击 Skill 提供的本地链接即可，不需要手动启动服务。

**语音识别停止了怎么办？**

页面不会无限自动重启麦克风。已确认文字会保留，可以重新点击麦克风或继续键盘输入。

## 评价会包含什么

面试结束后会给出结构化评价，重点是“本轮实际观察到的证据”，不会把没有考察到的内容强行打分。

评价通常包括：

1. 本轮面试概览和覆盖范围
2. 总分、已评分权重和评分可信度说明
3. 分维度评分
4. 明确优势
5. 主要问题与证据
6. 优先改进建议
7. 知识点纠错与学习建议，只有专业面试中确实有答错、没答上或关键机制缺失时才展示
8. 下一轮建议，包括推荐结论、依据和下一轮重点考察方向

提前结束面试时，也会基于已有证据评价；没考察到的部分会标记为“证据不足”。
如果在取得任何可评分证据前结束，Skill 不会用 0 分代替未知，而会将总分标记为“证据不足”，推荐结论限制为“待定”或“不建议”。

## 面试报告

评价结束后，Skill 会询问：

```text
是否为本轮面试生成面试报告？默认不生成。
```

只有你明确同意后，才会生成 HTML 报告。默认保存到当前授权工作目录下的 `interview-records/`。
如果同名报告已经存在，会自动追加 `-2`、`-3` 等后缀，不会覆盖旧报告。报告不依赖网络字体或其他外部资源，可以离线查看。

报告会保留：

- 面试基本信息和配置；
- 简历与职位描述的必要摘要；
- 实际问答过程；
- 分维度评分；
- 优势、问题、改进建议；
- 按需展示的知识点纠错与学习建议；
- 下一轮建议。

报告不会复制完整原始简历，只保留支持评价所需的摘要。用户原话会做 HTML 转义，避免代码片段或尖括号破坏页面。

## 知识点纠错与学习建议

这是专业面试报告中的按需模块。只有当本轮出现明显知识缺口时才展示，例如：

- 明确说不会或没接触过；
- 回答错误；
- 关键机制缺失；
- 对核心概念、边界条件或异常场景理解不完整。

每个知识点会包含：

- **本轮表现**：哪里没答上、答错或缺少关键机制；
- **正确理解**：给出相对完整但不过长的正确答案骨架；
- **面试中更好的回答**：下次应该如何组织回答；
- **学习入口**：建议优先补哪些概念。

这个模块和“主要问题与证据”不会重复展开。同一个知识点如果已经进入纠错模块，“主要问题与证据”里只保留一句诊断和影响。

## 常见配置

### 专业面主题数量

专业面主要按“主题”控制广度，而不是按总问题数。

| 面试轮次 | 默认主题数 | 可调整范围 |
|---|---:|---:|
| 专业一面 | 5 | 4～7 |
| 专业二面 / 资深员工面 | 6 | 5～7 |
| 专业三面 / 经理面 | 6 | 5～7 |
| 专业四面 / 总监面 | 5 | 4～6 |
| 专业五面 / 高管面 | 4 | 3～5 |

每个主题先提出一个主问题，主问题不计入追问次数。追问深度根据面试形式、面试官角色、轮次、目标职级和主题重要性动态确定：HR 初筛通常只需 1～2 次，专业一面通常 2～3 次，专业二面通常 3～4 次，经理或总监面会对关键主题进行更综合的 3～5 次深挖，高管面通常使用更少但信息密度更高的问题。建议下限不是强制要求，证据充分时可以提前结束；第 6 次追问始终是绝对上限。压力值主要改变挑战方式，不会自动增加问题数量。自我介绍和基于职位描述的收尾问答不计入主题数。

### 其他面试范围

| 类型 | 默认范围 |
|---|---|
| 人力资源初筛 | 5 个考察项 |
| 最终人力资源面 | 8 个考察项 |
| 行为面试 | 5 个行为能力或情境 |
| 跨部门面试 | 4 个协作主题或情境 |
| 薪资谈判 | 5 个谈判议题 |
| 编程或算法面试 | 3 个独立任务 |
| 系统设计面试 | 1 个完整案例 |
| 项目深挖 | 3 个项目或项目主题 |
| 作品集或方案答辩 | 2 个作品或方案 |
| 案例或业务分析 | 2 个独立案例 |

## 示例

下面的示例刻意覆盖不同岗位。编程和系统设计只是可选专项形式，不是专业面的默认含义。

```text
使用 $mock-interview，读取 ./resume.pdf，目标职位高级产品经理，进行经理面，重点考察产品决策和业务结果。
```

### 运营案例分析

```text
使用 $mock-interview，根据我的简历模拟一场区域运营经理案例分析面试，重点考察经营诊断和资源取舍。
```

### HR 面和薪资谈判

```text
使用 $mock-interview 进行最终人力资源面和薪资谈判演练，目标职位是大客户销售经理。
```

```text
使用 $mock-interview，根据我的简历模拟一场系统设计面试，目标职位是后端开发工程师，压力值 65。
```
