# Mock-Interview

`Mock-Interview` 是一个基于真实简历进行模拟面试的 Skill。它会先读取候选人的简历和目标岗位，再以真实面试官的方式逐轮提问、追问，并在结束后给出基于本轮回答的结构化评价。

它适合用来做：

- 专业一面、二面、经理面、总监面等技术或业务深挖；
- 人力资源初筛、最终 HR 面、薪资谈判演练；
- 行为面试、系统设计、算法面试、项目深挖、案例分析；
- 面试前查漏补缺，或面试后生成可复盘的 HTML 报告。

## 快速开始

你可以直接这样调用：

```text
使用 $mock-interview，根据我的简历进行一场后端开发专业二面。
```

如果简历在工作目录里，也可以指定路径：

```text
使用 $mock-interview，读取 ./求职材料/简历.pdf，模拟系统设计面试。
```

也可以提前指定配置：

```text
使用 $mock-interview 模拟高级产品经理的经理面，7 个主题，压力值 70，纯模拟模式，重点考察跨部门协作和业务取舍。
```

## 开始前需要什么

开始正式面试前，需要满足三个条件：

1. **有效简历**  
   支持 PDF、DOC、DOCX、RTF、PNG、JPG、JPEG、WEBP、TXT 和 Markdown。文件无法读取、内容为空或图片无法识别时，需要换一个可读取版本。

2. **目标职位**  
   目标职位必须由简历或用户明确给出，例如“后端开发工程师”“高级产品经理”“算法工程师”。Skill 不会只根据项目经历或技能栈猜岗位。

3. **面试轮次和类型**  
   例如“专业二面”“经理面”“最终 HR 面”“系统设计面试”“薪资谈判演练”。如果没说明，Skill 会先询问并给出合适选项。

职位描述、公司信息、题库、历史面试记录不是必需材料；如果提供了，Skill 会纳入提问和评价。

## 面试体验

正式开始前，Skill 会展示一份简短的本轮配置，包括候选人类型、面试类型、主题数量或考察项数量、压力值、面试官风格、语言、重点方向和反馈模式。

你需要确认反馈模式：

- **纯模拟**：面试过程中不点评，结束后统一反馈。
- **教练模式**：每次回答后给简短反馈，再继续面试。

正式面试开始后，会出现开始线：

```text
---------------------------- 面试开始 ----------------------------
```

开始线和结束线之间，输出会尽量保持真实面试感：不会显示题号、内部主题切换、已完成进度、评分意图或“这是一个追问”这类说明。

结束时会出现：

```text
---------------------------- 面试结束 ----------------------------
```

然后才会给出评价。

## 你可以随时控制面试

面试过程中可以直接说：

- `跳过`：换一个自然的问题。
- `暂停面试`：退出面试官角色，讨论面试过程。
- `继续面试`：恢复面试。
- `调整压力到 70`：即时调整压力值，范围 1～100。
- `结束面试`：立即停止提问，并基于已有回答评价。

## 评价会包含什么

面试结束后会给出结构化评价，重点是“本轮实际观察到的证据”，不会把没有考察到的内容强行打分。

评价通常包括：

1. 本轮面试概览和覆盖范围
2. 总分和评分可信度说明
3. 分维度评分
4. 明确优势
5. 主要问题与证据
6. 优先改进建议
7. 知识点纠错与学习建议，只有专业面试中确实有答错、没答上或关键机制缺失时才展示
8. 下一轮建议，包括推荐结论、依据和下一轮重点考察方向

提前结束面试时，也会基于已有证据评价；没考察到的部分会标记为“证据不足”。

## 面试报告

评价结束后，Skill 会询问：

```text
是否为本轮面试生成面试报告？
```

只有你明确同意后，才会生成 HTML 报告。默认保存到当前授权工作目录下的 `interview-records/`。

报告会保留：

- 面试基本信息和配置；
- 简历与职位描述的必要摘要；
- 实际问答过程；
- 分维度评分；
- 优势、问题、改进建议；
- 按需展示的知识点纠错与学习建议；
- 下一轮建议。

报告不会复制完整原始简历，只保留支持评价所需的摘要。用户原话会做 HTML 转义，避免代码片段或尖括号破坏页面。

## 知识点纠错与学习建议

这是专业面试报告中的按需模块。只有当本轮出现明显知识缺口时才展示，例如：

- 明确说不会或没接触过；
- 回答错误；
- 关键机制缺失；
- 对核心概念、边界条件或异常场景理解不完整。

每个知识点会包含：

- **本轮表现**：哪里没答上、答错或缺少关键机制；
- **正确理解**：给出相对完整但不过长的正确答案骨架；
- **面试中更好的回答**：下次应该如何组织回答；
- **学习入口**：建议优先补哪些概念。

这个模块和“主要问题与证据”不会重复展开。同一个知识点如果已经进入纠错模块，“主要问题与证据”里只保留一句诊断和影响。

## 常见配置

### 专业面主题数量

专业面主要按“主题”控制广度，而不是按总问题数。

| 面试轮次 | 默认主题数 | 可调整范围 |
|---|---:|---:|
| 专业一面 | 5 | 4～7 |
| 专业二面 / 资深员工面 | 6 | 5～7 |
| 专业三面 / 经理面 | 6 | 5～7 |
| 专业四面 / 总监面 | 5 | 4～6 |
| 专业五面 / 高管面 | 4 | 3～5 |

一个主题通常会追问 3～5 轮，最多 6 轮。自我介绍和基于职位描述的收尾问答不计入主题数。

### 其他面试范围

| 类型 | 默认范围 |
|---|---|
| 人力资源初筛 | 5 个考察项 |
| 最终人力资源面 | 8 个考察项 |
| 编程或算法面试 | 3 个独立任务 |
| 系统设计面试 | 1 个完整案例 |
| 项目深挖 | 3 个项目或项目主题 |
| 作品集或方案答辩 | 2 个作品或方案 |
| 案例或业务分析 | 2 个独立案例 |

## 项目结构

```text
mock-interview/
├── README.md
├── SKILL.md
├── agents/
│   └── openai.yaml
├── scripts/
│   ├── generate_report.py
│   ├── report-template.html
│   └── test_generate_report.py
└── references/
    ├── archive-format.md
    ├── interview-matrix.md
    └── scoring-rubrics.md
```

主要文件：

- `SKILL.md`：Skill 的执行流程和行为规则。
- `references/interview-matrix.md`：候选人类型、面试轮次、主题数量和默认配置。
- `references/scoring-rubrics.md`：评分方法和评价结构。
- `references/archive-format.md`：HTML 报告的 payload 结构和生成流程。
- `scripts/generate_report.py`：根据 JSON payload 生成 HTML 报告。
- `scripts/report-template.html`：HTML 报告模板。
- `scripts/test_generate_report.py`：报告生成脚本的回归测试。

## 开发与验证

修改报告模板或生成脚本后，运行：

```bash
python3 scripts/test_generate_report.py
```

测试会覆盖：

- HTML 转义；
- 重复区块渲染；
- 可选区块删除；
- 知识点纠错模块的按需展示；
- 报告导航和关键标题。
