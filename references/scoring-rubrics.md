# 评分规则

仅在结束面试时使用本参考资料。评价依据是本轮实际回答，而不是简历本身。

## 目录

1. 证据规则
2. 权重设计
3. 建议评分维度
4. 分数标准

## 证据规则

- 每项重要优势、风险和得分都要引用具体回答或本轮观察到的行为。
- 未考察或证据薄弱的维度标记为 **证据不足**。
- 不得在没有说明的情况下把缺少证据折算成低分。
- 如果证据不足以支持可靠的数字总分，给出暂定总分并说明覆盖范围的限制。
- 用户没有解释清楚前，将前后矛盾视为尚未解决的证据问题。
- 明确说明结论是基于模拟面试的建议，而不是客观的招聘决定。

## 权重设计

根据候选人类型、目标职位、面试轮次和实际考察范围动态选择评分维度。所有维度的权重之和必须恰好为 100%。

对于实习生和应届生，提高专业基础、学习能力、潜力和迁移能力的权重。对于社招和高级岗位，提高成果、专业深度、决策、影响力和业务价值的权重。

专业面可以从以下维度中选择：

- 岗位匹配度；
- 专业基础；
- 专业深度；
- 问题分析与解决；
- 项目真实性和个人贡献；
- 方案取舍质量；
- 沟通与协作；
- 学习与成长能力。

人力资源面可以从以下维度中选择：

- 求职动机；
- 沟通表达；
- 经历一致性；
- 行为案例质量；
- 稳定性和职业规划；
- 薪资与到岗匹配度；
- 压力下的职业表现。

必要时增加岗位特有维度。不得仅为填满模板，对当前轮次有意未考察的维度强行打分。

### 高级经理、总监或高管面

## 分数标准

对每个有证据的维度使用以下标准：

- **90～100 分**：证据突出、具体且一致；充分体现原理、边界、取舍和影响；
- **75～89 分**：证据扎实，仅有少量缺口；基本能够证明达到目标级别并可独立胜任；
- **60～74 分**：证据有好有坏；部分表现合格，但深度、清晰度或一致性存在明显缺口；
- **40～59 分**：证据薄弱；反复出现模糊、理解浅或矛盾尚未解决的情况；
- **0～39 分**：已有证据显示明显不符合目标级别，或存在严重风险。

按照各已评分维度的权重计算加权总分。清楚标记因证据不足而未评分的维度，并说明这对结论可信度的限制。

### 总分等级

| 总分范围 | 等级 | 含义 |
|---|---|---|
| 90～100 | 卓越 | 多数维度表现突出，强烈建议通过 |
| 75～89 | 良好 | 整体表现扎实，建议通过 |
| 60～74 | 合格 | 存在明显的优势和不足，需要综合判断 |
| 40～59 | 待提升 | 多个维度存在风险，不建议通过 |
| 0～39 | 不通过 | 基本面存在严重问题 |

等级仅作为参考框架，最终建议仍须结合具体维度表现和岗位要求综合判断。

## 必须输出的内容

始终包含：

1. **本轮面试概览**：目标职位、候选人类型、轮次和类型、配置，以及专业面实际覆盖的主题或其他面试实际覆盖的考察项、任务与案例。专业面说明主题覆盖范围，不以总问题数作为完成度指标。
2. **总分（0～100）**：加权总分，以及简短的可信度或覆盖范围说明。
3. **分维度评分**：维度、权重、得分或“证据不足”，以及简明依据。
4. **明确优势**：每项都对应具体回答或行为；只写本轮已经被证明的强项，不混入待改进建议。
5. **主要问题与证据**：合并呈现风险、不足、矛盾、模糊点、知识缺口和证据不足。每项都说明问题类型、具体证据和影响，不要把同一问题拆到多个模块重复表述。已经进入“知识点纠错与学习建议”的内容，在本模块中只保留一句诊断和影响，不展开正确理解或学习建议。
6. **优先改进建议**：合并呈现最值得改进的 3～5 项内容和关键问题的更优回答思路。每项都绑定一个具体问题，说明优先级、应补充的证据或知识点、可执行改法和更好的回答结构；不得编造经历。
7. **知识点纠错与学习建议**：仅在专业面试中按需输出；如果本轮存在候选人没答上、答错、关键机制缺失或核心概念理解错误的知识点，单独列出本模块。每个知识点包含本轮表现、正确理解、面试中更好的回答方式和学习入口；其中“正确理解”可以稍微展开，帮助用户知道正确答案，但不要写成大篇幅教程。如果没有这类知识点，不输出本模块。
8. **下一轮建议**：合并呈现是否建议进入下一轮和下一轮可能重点考察的方向。推荐结论必须且只能选择“强烈建议”“建议”“待定”“不建议”之一，并说明依据；随后给出下一轮可能重点验证的主题、风险或能力边界。

用户提前结束时，立即根据已有证据输出上述结构，并将未覆盖的部分标记为 **证据不足**。

> 是否为本轮面试生成面试报告？
