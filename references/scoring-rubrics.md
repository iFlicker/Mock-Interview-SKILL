# 评分规则

本参考资料分两次使用：配置确认后、正式面试开始前建立内部评分蓝图；结束面试时按照同一蓝图评分。评价依据是本轮实际回答，而不是简历本身。

## 评分蓝图

在正式面试开始前，根据候选人类型、目标职位和职级、面试阶段、面试官角色、面试形式及计划范围，确定评分维度、权重和预期证据来源。所有维度权重之和必须恰好为 100%。

面试开始后冻结维度和权重，直至本轮评价完成。不得因为某个维度没有问到、回答不理想、难以评分或用户调整范围而删除该维度、降低其权重或换成更有利的维度。用户在面试中调整压力、重点、题目范围或覆盖数量时，只调整提问计划并记录原因，不修改评分蓝图；如果用户改变目标职位或面试形式，以至于原蓝图已经不适用，应暂停并说明需要结束本轮、按新配置开始一次新面试，不得在同一评分会话中换蓝图。

评分覆盖率以冻结蓝图为分母：`评分覆盖率 = 已取得数字得分的维度权重之和`。未考察或证据不足的维度保留原权重，得分标记为“证据不足”。

## 证据规则

- 评分只能使用 `session-state.md` 中 `evidence_ledger` 已记录且能关联到正式问题的证据；不得在评价阶段根据印象补建更有利的事实。
- 每项重要优势、风险和得分都要引用具体回答或本轮观察到的行为。
- 简历只用于背景理解和选择问题。除非候选人在本轮回答中确认或展开相关内容，否则简历陈述不能直接作为得分或优势的证据。
- 未考察或证据薄弱的维度标记为 **证据不足**。
- 不得在没有说明的情况下把缺少证据折算成低分。
- 只要至少一个维度取得数字得分，就按本文件公式给出数字总分，并在覆盖率不足时明确标记为暂定结果；只有评分覆盖率为 0% 时，总分才标记为“证据不足”。
- 用户没有解释清楚前，将前后矛盾视为尚未解决的证据问题。
- 只有分类为 `substantive_conflict` 且状态为 `unresolved` 的矛盾可以作为负面证据；表述差异、上下文缺失和已经解决的矛盾不得继续扣分。
- 每条关键证据区分为 **独立回答** 或 **教练提示后回答**。教练模式下的即时反馈不得作为候选人证据；提示后回答可以用于诊断学习效果，但评分时优先依据独立回答，并说明提示对结论可信度的影响。
- `insufficient` 表示无法判断，不等于负面表现；只有 `negative` 或多条一致的 `partial` 证据才能支持明确扣分。证据置信度为低时必须降低结论确定性。
- 明确说明结论是基于模拟面试的建议，而不是客观的招聘决定。

## 权重设计

根据候选人类型、目标职位和职级、面试阶段、面试官角色、面试形式和计划考察范围选择评分维度，并在面试前冻结。所有维度的权重之和必须恰好为 100%。

对于实习生和应届生，提高专业基础、学习能力、潜力和迁移能力的权重。对于社招和高级岗位，提高成果、专业深度、决策、影响力和业务价值的权重。

专业面可以从以下维度中选择：

- 岗位匹配度；
- 专业基础；
- 专业深度；
- 问题分析与解决；
- 项目真实性和个人贡献；
- 方案取舍质量；
- 沟通与协作；
- 学习与成长能力。

人力资源面可以从以下维度中选择：

- 求职动机；
- 沟通表达；
- 经历一致性；
- 行为案例质量；
- 稳定性和职业规划；
- 谈判沟通与条件匹配；
- 压力下的职业表现。

必要时增加岗位特有维度。蓝图中可以保留计划考察但最终证据不足的维度，不能为了填满模板而给出数字分数。没有真实岗位预算、薪资范围或到岗要求时，“谈判沟通与条件匹配”只评价表达、依据、总包理解、优先级和取舍，不得根据期望金额高低或个人让步程度给分。

## 分数标准

对每个有证据的维度使用以下标准：

- **90～100 分**：证据突出、具体且一致；充分体现原理、边界、取舍和影响；
- **75～89 分**：证据扎实，仅有少量缺口；基本能够证明达到目标级别并可独立胜任；
- **60～74 分**：证据有好有坏；部分表现合格，但深度、清晰度或一致性存在明显缺口；
- **40～59 分**：证据薄弱；反复出现模糊、理解浅或矛盾尚未解决的情况；
- **0～39 分**：已有证据显示明显不符合目标级别，或存在严重风险。

所有维度的权重，包括证据不足的维度，仍须合计为 100%。总分只对已有数字得分的维度重新归一化计算：`总分 = Σ（维度得分 × 维度权重）÷ Σ（已评分维度权重）`，四舍五入为整数。同时报告已评分维度的权重覆盖率。

如果没有任何维度取得足够证据，分母为零，不计算数字总分；总分显示为 **证据不足**，评分覆盖率为 0%，推荐结论只能为“待定”或“不建议”。不得用 0 分代替无法评分。

### 总分等级

| 总分范围 | 等级 | 含义 |
|---|---|---|
| 90～100 | 卓越 | 多数维度表现突出，强烈建议通过 |
| 75～89 | 良好 | 整体表现扎实，建议通过 |
| 60～74 | 合格 | 存在明显的优势和不足，需要综合判断 |
| 40～59 | 待提升 | 多个维度存在风险，不建议通过 |
| 0～39 | 不通过 | 基本面存在严重问题 |

等级仅作为参考框架，最终建议仍须结合具体维度表现和岗位要求综合判断。

## 推荐结论

推荐结论必须且只能选择“强烈建议”“建议”“待定”“不建议”之一，并遵守以下约束：

| 条件 | 允许的最高结论 |
|---|---|
| 评分覆盖率低于 60%，或没有数字总分 | 待定 |
| 总分 0～59 | 不建议 |
| 总分 60～74 | 待定 |
| 总分 75～89，评分覆盖率至少 60%，且没有 P0 风险 | 建议 |
| 总分 90～100，评分覆盖率至少 80%，关键维度均有独立回答证据，且没有 P0 风险 | 强烈建议 |

“最高结论”表示可以根据风险选择更保守的结论。存在诚信、岗位硬门槛、核心能力或职业边界方面的 P0 风险时，无论总分多少，结论最高为“不建议”；但必须给出本轮具体证据，不得仅凭怀疑设置 P0。教练模式下，如果关键维度主要依赖提示后回答，即使总分较高，结论最高为“待定”。

## 必须输出的内容

始终包含：

1. **本轮面试概览**：目标职位、候选人类型、轮次和类型、配置，以及专业面实际覆盖的主题或其他面试实际覆盖的考察项、任务与案例。专业面说明主题覆盖范围，不以总问题数作为完成度指标。
2. **总分与覆盖率**：存在可评分维度时输出 0～100 加权总分；没有可评分维度时输出“证据不足”，不得输出虚构数字。同时输出已评分权重覆盖率和简短的可信度说明。
3. **分维度评分**：维度、权重、得分或“证据不足”，以及简明依据。
4. **明确优势**：每项都对应具体回答或行为；只写本轮已经被证明的强项，不混入待改进建议。
5. **主要问题与证据**：合并呈现风险、不足、矛盾、模糊点、知识缺口和证据不足。每项都说明问题类型、具体证据和影响，不要把同一问题拆到多个模块重复表述。已经进入“知识点纠错与学习建议”的内容，在本模块中只保留一句诊断和影响，不展开正确理解或学习建议。
6. **优先改进建议**：合并呈现最值得改进的 3～5 项内容和关键问题的更优回答思路。每项都绑定一个具体问题，说明优先级、应补充的证据或知识点、可执行改法和更好的回答结构；不得编造经历。
7. **知识点纠错与学习建议**：仅在专业面试中按需输出；如果本轮存在候选人没答上、答错、关键机制缺失或核心概念理解错误的知识点，单独列出本模块。每个知识点包含本轮表现、正确理解、面试中更好的回答方式和学习入口；其中“正确理解”可以稍微展开，帮助用户知道正确答案，但不要写成大篇幅教程。如果没有这类知识点，不输出本模块。
8. **下一轮建议**：合并呈现是否建议进入下一轮和下一轮可能重点考察的方向。按照本文件“推荐结论”约束选择结论并说明依据；随后给出下一轮可能重点验证的主题、风险或能力边界。

用户提前结束时，立即根据已有证据输出上述结构，并将未覆盖的部分标记为 **证据不足**。

评价完成后，返回 `SKILL.md` 中定义的报告授权步骤。本文件只定义评分与推荐结论，不重复定义文件生成行为。
