# 网页语音面试平台限制

## 宿主差异

| 宿主 | 官方 MCP 方式 | 当前会话适配 | 客观限制 |
|---|---|---|---|
| Codex | stdio 或 Streamable HTTP；桌面、CLI、IDE 共享配置 | 启动同一 MCP 服务，并在当前任务使用本地控制入口 | 新增 MCP 配置后需要客户端重启，不能把本轮临时服务热注入已运行任务 |
| Claude Code | `claude mcp add` 支持本地、项目和用户作用域 | 自动注册唯一名称的本地 HTTP MCP，结束时自动移除；失败时使用本地控制入口 | 企业策略或当前版本可能要求授权或刷新工具清单 |
| Cursor | `mcp.json`；扩展 API 可动态注册 | 扩展 API 不在 Skill 上下文时使用本地控制入口 | 普通 Agent/Skill 不能冒充扩展调用动态注册 API，配置文件通常需要重新加载 |
| OpenCode | `opencode.json` 的本地或远程 MCP；`opencode mcp add` | 使用同一 MCP 服务的本地控制入口 | `mcp add` 是交互式流程，配置变更能否被当前会话热加载取决于宿主版本 |
| 其他宿主 | 取决于其 MCP stdio/HTTP 实现 | 优先使用 stdio MCP；否则使用本地控制入口 | 不声称所有兼容客户端都支持运行时注册或相同超时 |

本地控制入口不是另一套协议：它通过 localhost 调用完全相同的 MCP `tools/call`，用于绕过“宿主不能在当前会话刷新工具清单”的生命周期限制，不绕过面试状态机或安全校验。

所有当前支持的宿主都存在同一项生命周期限制：本地 MCP 服务可以保存网页事件，但不能主动唤醒已经结束或进入 idle 的 Agent 回合。Skill 因此在每个网页问题推送后使用持续等待进程持有当前回合，进程内部仍以最长 25 秒的 MCP 调用轮询。候选人沉默或短等待超时不应结束任务；若宿主强制中止长时间运行的 Agent，网页会根据心跳显示“无法连接”，用户可返回当前应用恢复或切换为文字面试。

不同宿主对长命令使用 cell、session 或其他续读句柄，但统一要求是恢复原句柄，而不是再次启动等待命令。runtime 的单等待者租约是跨宿主的最后一道防护；它阻止并发等待，却不能替宿主自动恢复一个已经被 Agent 遗忘的 cell，因此宿主适配说明仍必须明确各自的续读方式。

## 浏览器差异

- `speechSynthesis` 的声音、中文断句和 `onboundary` 支持由浏览器、操作系统与具体系统声音共同决定；没有可靠边界时页面只显示朗读状态。
- 原生 `SpeechRecognition` 并非所有浏览器都提供；部分实现会使用浏览器厂商在线服务，无法保证完全本地或离线。
- 麦克风权限提示、设备切换行为和后台页策略由浏览器控制。
- 自动打开浏览器可能被系统策略阻止；用户仍可点击返回的 localhost URL。

## 环境限制

云端 Agent、远程容器、SSH 和无图形桌面环境默认不启动本地语音面试室，因为服务端的 localhost 通常不是用户电脑的 localhost。此时立即降级为文字面试，保留已经确认的配置。
